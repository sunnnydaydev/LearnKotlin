###### 权限修饰符

```java


/*
*   权限修饰符：
* 1、public ：kotlin默认（java不写任何修饰符默认为包访问权限）
* 2、protected：  java中同一包中可以访问，kotlin中只能在同一类或者继承的子类中使用
* 3、private： java 与kotlin使用一样
*
* 4、包访问权限：java 默认的，不写修饰符时默认为包访问权限。kotlin没有次概念。
* 5、internal：kotlin新增的关键字，模块访问权限。模块内部可见。一个模块就是一起一组编译的kotlin文件
*
* ps：internal与包访问 权限对比：
* 1、对细节真正的隐藏
* 2、使用java时，外部代码可以将类定义在你的包中，从而访问你的包私有声明的权限。
* 3、kotlin允许你在顶层声明使用private可见性，这些声明只在声明他们的文件中可见。
*
*
*
* */
```
